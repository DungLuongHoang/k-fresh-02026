# Code of Conduct

## AI QA Training – Community Standards

Thank you for contributing to this project.

This repository is focused on modern Quality Engineering practices, AI-assisted testing, automation frameworks, and collaborative learning. To maintain a productive and professional environment, all contributors are expected to follow this Code of Conduct.

---

# Our Commitment

We are committed to providing a welcoming, respectful, and inclusive environment for everyone participating in this project.

Contributors, maintainers, and participants should:

- Collaborate professionally
- Respect different perspectives and experiences
- Provide constructive feedback
- Encourage knowledge sharing and continuous learning
- Support inclusive and positive communication

---

# Expected Behavior

Examples of positive behavior include:

- Using respectful and professional language
- Helping others learn and grow
- Giving constructive code review feedback
- Sharing knowledge openly
- Writing clean, maintainable, and documented code
- Validating AI-generated outputs before submission
- Following project standards and contribution guidelines

---

# Unacceptable Behavior

The following behaviors are unacceptable:

- Harassment or discrimination of any kind
- Personal attacks or insulting comments
- Trolling or intentionally disruptive behavior
- Sharing confidential or private information
- Spam, self-promotion, or irrelevant discussions
- Submitting malicious, insecure, or plagiarized code
- Abusing AI tools to generate low-quality or misleading contributions

---

# AI Usage Guidelines

This repository encourages responsible AI-assisted development.

Contributors may use AI tools for:

- Test generation
- Code suggestions
- Documentation drafting
- Bug analysis
- Automation enhancement

However:

- Contributors remain responsible for validating all generated content
- AI-generated code must meet project quality standards
- Security, reliability, and maintainability must always be reviewed manually
- Generated content should not violate copyrights or licenses

---

# Contribution Standards

Before submitting contributions:

- Ensure code passes linting and tests
- Follow the repository structure and naming conventions
- Keep pull requests focused and clearly documented
- Add or update tests when applicable
- Write meaningful commit messages
- Maintain backward compatibility when possible

---

# Enforcement

Project maintainers may take action against behavior that violates this Code of Conduct, including:

1. Warning
2. Removal of comments or contributions
3. Temporary restriction from participation
4. Permanent ban from the repository or community

Enforcement decisions are made to protect the health and professionalism of the project.

---

# Reporting Issues

If you experience or witness unacceptable behavior, please report it through:

- GitHub Issues
- GitHub Discussions
- Repository maintainers

All reports will be handled respectfully and confidentially.

---

# Scope

This Code of Conduct applies to:

- GitHub Issues
- Pull Requests
- Discussions
- Code Reviews
- Project Documentation
- Community Communication Channels
